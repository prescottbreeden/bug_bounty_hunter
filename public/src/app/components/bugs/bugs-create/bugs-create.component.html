Guidelines

Defining an Error

When you are posting a bug, try to ascertain which part of your error message is the most pertinent and descriptive.

Posting a Traceback

Your traceback refers to the block of code that references each line of code that failed because of the error. This output can be extremely helpful when debugging to get additional info but takes practice to filter what's noise and what's integral. If you are unsure what your traceback is, just copy the whole error message and paste it here.

Adding a Message

This is a really good place to mention the ways you have already tried to solve your bug. Not only will this reduce thread redundancy but will also offer an opportunity for you to rubber-duck-debug in case you forgot to try something. If there is any additional info that you think might be helpful you can add it here (e.g. code snippets you think might be the cause, questions about the error message itself, etc.).
